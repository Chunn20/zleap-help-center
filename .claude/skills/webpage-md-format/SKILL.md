# webpage-md-format

统一 Fumadocs / MDX 帮助文档页面格式的 Skill。

## 用途

根据标准格式规范改写帮助文档页面，统一页面结构、标题层级、组件用法、段落密度和表达方式。

## 触发场景

当用户使用以下任一表述时，自动使用此 Skill：

- "参考标准格式改页面"
- "参考标准 md 改页面格式"
- "统一帮助文档页面样式"
- "把某个页面改成标准格式"
- "应用标准格式"
- "格式标准化"
- "按标准格式改写"
- "按标准格式整理"
- "统一页面风格"
- "按帮助中心格式优化"

## 唯一格式标准

**`references/standard-page.md` 是唯一格式标准。**

### 重要原则

1. **不参考旧页面格式**：不使用项目中已有页面（如 quick-start.mdx、information-management.mdx、subscribe-shared.mdx、about.mdx 等）作为隐性标准。
2. **不使用现有页面作为参考**：即使这些页面看起来"格式正确"，也可能存在历史遗留问题。
3. **standard-page.md 只作为格式规范**：它是格式和风格参考，不是内容来源。
4. **只学习结构和风格**：改写时只学习页面结构、标题层级、组件组织、段落密度、提示区样式和表达方式。
5. **不照搬具体内容**：不要照搬 standard-page.md 中的具体文字、示例或产品描述。

## 改写原则

### 必须保留的内容

1. **frontmatter**：title、description 等元数据
2. **事实信息**：产品功能、操作步骤、技术细节
3. **图片路径**：所有图片的 src 路径和 alt 文本
4. **链接 URL**：所有超链接的 href 属性
5. **代码块**：命令、参数、配置项、代码示例
6. **产品功能含义**：不得改变产品功能的核心含义

### 必须修改的内容

1. **CSS 样式**：按 standard-page.md 中的标准 CSS 调整
   - H1: 36px, margin-bottom: 0
   - H2: 26px/600, margin-top: 60px, margin-bottom: 24px
   - H3: 20px/600, margin-top: 32px, margin-bottom: 16px
   - 正文: 16px/400, line-height: 26px
   - 颜色: #1A1D23

2. **灰色卡片（核心概念）**：改为普通段落（16px/400/26px）

3. **黄色提示框**：改为简单的 icon + 文字，无背景
   ```jsx
   <div style={{ display: 'flex', gap: '12px', alignItems: 'center', margin: '32px 0' }}>
     <div style={{ flexShrink: 0 }}>
       <Lightbulb className="h-5 w-5" style={{ color: '#FF8800' }} />
     </div>
     <p style={{ fontSize: '16px', fontWeight: 400, color: '#1A1D23', lineHeight: '26px', margin: 0 }}>
       <strong>提示：</strong>提示内容文字
     </p>
   </div>
   ```

4. **卡片网格**：改为"典型应用场景"格式（手写 h3 + 橙色关键词）

5. **分隔线**：删除 `---` 分隔线

6. **段落合并**：语义连贯的多段落可以合并

7. **内容精简**：删除重复、空泛、解释过度的句子

### CSS 处理规则

**不在每个页面重复写 CSS。** 只按标准格式调整页面结构和 MDX 写法。

如果页面已有 `<style>` 标签，检查是否符合标准格式：
- 符合标准：保留不动
- 不符合标准：按 standard-page.md 中的标准 CSS 修正

如果页面没有 `<style>` 标签，且全局 CSS 已覆盖标准格式，则不需要添加。

### 冲突处理规则

1. **standard-page.md 与当前页面冲突**：以 standard-page.md 为准
2. **standard-page.md 与项目全局 CSS 冲突**：以全局 CSS 为准
3. **发现新的格式冲突**：先问用户，不要自行决定

### 内容不足时的处理

如果目标页面内容不足以完成改写，使用 `<!-- TODO: 补充内��� -->` 标注，不要编造内容。

## 输出要求

1. **输出完整页面**：直接修改目标文件，或输出完整可替换的页面内容
2. **简要说明**：修改后简单说明改了哪些结构（2-3 句话）
3. **不逐行解释**：不需要逐行解释每个修改点

## 示例说明格式

```
已完成格式标准化：

1. CSS 样式：H2 改为 26px/600，H3 改为 20px/600，正文行高改为 26px
2. 灰色卡片改为普通段落
3. 黄色提示框改为 icon + 文字格式
4. 删除 3 处分隔线
5. 合并 2 组语义连贯的段落
```

## 注意事项

1. **不改变产品功能含义**：改写时只调整表达方式，不改变功能描述的核心含义
2. **保留代码块完整性**：命令、参数、配置项不得擅自改写
3. **保留图片和链接**：所有图片路径、alt 文本、链接 URL 必须保持不变
4. **不编造内容**：内容不足时用 TODO 标注，不要编造
5. **遇到疑问先问**：如果遇到格式冲突或不确定的情况，先问用户，不要自行决定
