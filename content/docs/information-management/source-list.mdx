## 核心价值：统一数据入口

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, margin: '0 0 24px 0' }}>
  Zleap 的核心是「用 AI 理解你的私有数据，生成精准回答」，而信息源列表的首要价值就是<strong style={{ color: '#FF8A00', fontWeight: 600 }}>「统一数据入口」</strong>——把分散在飞书、文档、网页、代码库、企业微信等各处的数据，统一归集到一个平台管理，让 AI 能一站式读取所有信息，不再需要在多个系统间反复切换。
</p>

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, margin: '0 0 16px 0' }}>
  基于这个统一入口，信息源列表还提供了三大关键能力：
</p>

<ul style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, margin: '0 0 40px 0', paddingLeft: '20px' }}>
  <li><strong>知识 freshness 保障</strong>：通过同步状态、同步频率配置，保证 AI 的知识库永远是最新的，不会用过时信息回答</li>
  <li><strong>权限与安全管控</strong>：通过启用 / 停用、权限划分，严格控制 AI 能访问的数据范围，避免敏感信息泄露</li>
  <li><strong>效率提升</strong>：一站式管理所有数据，不用在多个平台之间切换，大幅降低 AI 知识库的维护成本</li>
</ul>

## 事件表格功能


### 什么是「事件表格」？

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, margin: '12px 0 16px 0' }}>
  事件表格是 SAG（SQL 驱动的 RAG 引擎）将非结构化数据（如文档、聊天记录）通过 AI 预处理后，转化而成的结构化数据库（关系型的表单数据），后续 SAG 可以根据此数据库构建事件之间的联系。
</p>

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, margin: '12px 0 20px 0' }}>
  该模式支持用户以事件为主视角快速浏览、筛选和定位信息来源下的所有事件的功能，以提升排查与分析效率。在同一页面内串联「事件—信息来源（源文档）—原文内容」三层信息，减少页面跳转和心智切换成本，保证查看路径清晰、一致。
</p>

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, margin: '32px 0 12px 0' }}>
  <strong>提取前 - 原始零散的信息</strong>
</p>

<div style={{ margin: '24px auto', borderRadius: '12px', overflow: 'hidden', border: '1px solid rgba(0, 0, 0, 0.04)', maxWidth: '100%', width: '100%', height: '400px', padding: '24px', display: 'flex', alignItems: 'center', justifyContent: 'center', background: '#FAFAFA', cursor: 'pointer', transition: 'transform 0.2s' }}>
