## 操作步骤

### 前置条件：在 GitLab 中创建 OAuth 应用

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '16px' }}>
  在 Zleap 中配置 GitLab 之前，需要先在 GitLab 中创建一个 OAuth 应用程序。
</p>

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '16px' }}>
  登录您的 GitLab 账号（GitLab.com 或自托管实例），进入【User Settings】-【Applications】，点击【Add new application】创建新应用。
</p>

<div style={{ margin: '24px auto', borderRadius: '12px', overflow: 'hidden', border: '1px solid rgba(0, 0, 0, 0.04)', maxWidth: '100%', width: '100%', height: '400px', padding: '24px', display: 'flex', alignItems: 'center', justifyContent: 'center', background: '#FAFAFA', cursor: 'pointer', transition: 'transform 0.2s' }}>
  <img src="/images/information-management/create-private/gitlab/create-app.gif" alt="创建GitLab OAuth应用" style={{ maxWidth: '100%', maxHeight: '100%', width: 'auto', height: 'auto', display: 'block', objectFit: 'contain' }} />
</div>

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '12px' }}>填写应用信息：</p>

<ul style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.8, paddingLeft: '20px', marginBottom: '16px' }}>
  <li><strong>Name：</strong>应用名称，例如 "Zleap Integration"</li>
  <li><strong>Redirect URI：</strong>回调地址，从 Zleap 配置页面复制</li>
  <li><strong>Scopes：</strong>勾选所需权限（至少需要 read_user、read_repository）</li>
</ul>



<div style={{ background: '#FFF9F0', border: '1px solid #FFE9CC', borderRadius: '12px', padding: '20px 24px', margin: '32px 0' }}>
  <p style={{ fontSize: '14px', color: '#1d1d1f', lineHeight: 1.7, margin: 0, display: 'flex', alignItems: 'center', gap: '8px' }}>
    <CheckCircle className="h-5 w-5" style={{ color: '#FF8A00', flexShrink: 0 }} />
    创建成功后，GitLab 会生成 <strong>Application ID</strong> 和 <strong>Secret</strong>，请妥善保存这两个值。
  </p>
</div>

### 在 Zleap 中配置 GitLab

### 步骤1：进入 GitLab 配置页面

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '16px' }}>
  在 Zleap 中进入【信息管理】-【创建信息源】-【GitLab】。
</p>



### 步骤2：选择 GitLab 实例类型

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '16px' }}>
  根据您使用的 GitLab 类型选择：GitLab.com（使用官方云服务）或 Self-hosted GitLab（使用企业自托管实例）。
</p>


### 步骤3：填写 GitLab 实例地址（仅自托管）

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '16px' }}>
  如果选择了 Self-hosted GitLab，需要填写您的 GitLab 实例地址。
</p>


### 步骤4：复制回调地址

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '16px' }}>
  在 Zleap 配置页面中，复制显示的【Redirect URI】（回调地址）。
</p>



### 步骤5：填写 Application ID 和 Secret

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '16px' }}>
  将从 GitLab OAuth 应用中获取的凭证填入 Zleap。
</p>

<ul style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.8, paddingLeft: '20px', marginBottom: '16px' }}>
  <li><strong>Application ID：</strong>粘贴 GitLab 生成的 Application ID</li>
  <li><strong>Secret：</strong>粘贴 GitLab 生成的 Secret</li>
</ul>



### 步骤6：完成授权

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '16px' }}>
  点击【去授权】按钮，页面会自动跳转到 GitLab 授权页面。确认授权 Zleap 访问您的 GitLab 数据，授权成功后自动返回 Zleap，即完成 GitLab 接入。
</p>


<div style={{ background: '#FFF9F0', border: '1px solid #FFE9CC', borderRadius: '12px', padding: '20px 24px', margin: '32px 0' }}>
  <p style={{ fontSize: '14px', color: '#1d1d1f', lineHeight: 1.7, margin: 0, display: 'flex', alignItems: 'center', gap: '8px' }}>
    <CheckCircle className="h-5 w-5" style={{ color: '#FF8A00', flexShrink: 0 }} />
    配置完成后，Zleap 就可以访问您授权的 GitLab 仓库信息了！
  </p>
</div>

## 关键说明

### 关于权限范围（Scopes）

<p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '16px' }}>
  在 GitLab 中创建 OAuth 应用时，需要选择合适的权限范围：
</p>

<ul style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.8, paddingLeft: '20px', marginBottom: '16px' }}>
  <li><strong>read_user：</strong>读取用户基本信息（必需）</li>
  <li><strong>read_repository：</strong>读取仓库代码和文件（推荐）</li>
  <li><strong>read_api：</strong>读取 API 数据，包括 Issue、MR 等（推荐）</li>
</ul>

<div style={{ background: '#FFF9F0', border: '1px solid #FFE9CC', borderRadius: '12px', padding: '20px 24px', margin: '32px 0' }}>
  <p style={{ fontSize: '14px', color: '#1d1d1f', lineHeight: 1.7, margin: 0, display: 'flex', alignItems: 'center', gap: '8px' }}>
    <AlertCircle className="h-5 w-5" style={{ color: '#FF8A00', flexShrink: 0 }} />
    <strong>安全提示：</strong>只授予 Zleap 必要的权限，避免授予写入权限（如 write_repository、api）
  </p>
</div>

### 常见错误排查

<div style={{ background: '#FFFFFF', border: '1px solid rgba(0, 0, 0, 0.08)', borderRadius: '16px', padding: '20px', marginBottom: '16px' }}>
  <div style={{ display: 'flex', alignItems: 'center', gap: '8px', marginBottom: '12px' }}>

    <p style={{ fontSize: '17px', fontWeight: 600, color: '#1d1d1f', margin: 0 }}>错误 1：Redirect URI mismatch</p>
  </div>
  <p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '8px', paddingLeft: '32px' }}><strong>原因：</strong>GitLab OAuth 应用中配置的回调地址与 Zleap 提供的不一致</p>
  <p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, margin: 0, paddingLeft: '32px' }}><strong>解决：</strong>检查并确保 GitLab 应用配置中的 Redirect URI 与 Zleap 显示的完全一致（包括协议、域名、路径）</p>
</div>

<div style={{ background: '#FFFFFF', border: '1px solid rgba(0, 0, 0, 0.08)', borderRadius: '16px', padding: '20px', marginBottom: '16px' }}>
  <div style={{ display: 'flex', alignItems: 'center', gap: '8px', marginBottom: '12px' }}>

  </div>
  <p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, marginBottom: '8px', paddingLeft: '32px' }}><strong>原因：</strong>Application ID 或 Secret 填写错误</p>
  <p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, margin: 0, paddingLeft: '32px' }}><strong>解决：</strong>重新从 GitLab OAuth 应用页面复制 Application ID 和 Secret，确保没有多余的空格或字符</p>
</div>

<div style={{ background: '#FFFFFF', border: '1px solid rgba(0, 0, 0, 0.08)', borderRadius: '16px', padding: '20px', marginBottom: '16px' }}>

  <p style={{ fontSize: '17px', color: '#1d1d1f', lineHeight: 1.7, margin: 0, paddingLeft: '32px' }}><strong>解决：</strong>确认 GitLab 实例地址正确，且 Zleap 服务器可以访问该地址（检查防火墙、VPN 等网络配置）</p>
</div>
