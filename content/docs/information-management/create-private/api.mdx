  一、什么是 API？
</h2>

<div className="api-section">
  <p className="api-intro">

  </p>

  <p className="api-intro">
    从其他平台获取数据到智跃 Zleap 里，API 就是双方提前约定好的 "快递路线"：
  </p>

  <ul style={{ fontSize: '14px', color: 'var(--text-secondary, #5D5D5D)', lineHeight: 1.8, paddingLeft: '20px', marginBottom: '16px' }}>
    <li><strong>API 请求地址</strong> = 快递要送达的目的地</li>
    <li><strong>参数</strong> = 快递面单上的 "备注信息"（告诉对方你要什么类型、什么范围的数据）</li>
    <li><strong>请求方式（如 GET）</strong> = 快递的 "取件规则"（比如是 "上门取件" 还是 "网点自提"）</li>
  </ul>

  <p className="api-intro">

  FAQs

<div className="api-section">
  <h4 style={{ fontSize: '16px', fontWeight: 600, color: 'var(--text-primary, #0D131A)', marginBottom: '12px' }}>
    为什么要设置超时时间？
  </h4>

  <p className="api-intro">
    你发了一个 "取件请求"（API 请求），并约定最多等 30 秒。如果快递员（对方系统）30 秒内没把数据送过来，你就会判定这次取件失败，不再继续等下去。
  </p>

  <ul style={{ fontSize: '14px', color: 'var(--text-secondary, #5D5D5D)', lineHeight: 1.8, paddingLeft: '20px', marginBottom: '16px' }}>
    <li><strong>🚫 避免系统卡住</strong>：如果对方系统出现故障、网络中断，没有超时时间的话，你的系统可能会无期限等待，导致界面卡住、资源浪费。</li>
    <li><strong>⏱️ 保证体验与效率</strong>：合理的超时时间（如 30 秒）能让系统及时判定请求失败，你可以快速排查问题（比如检查网络、联系对方管理员），而不是让用户一直干等。</li>
    <li>
      <strong>⚖️ 平衡稳定性和成功率</strong>：
      <ul style={{ marginTop: '8px', paddingLeft: '20px' }}>
        <li>时间太短（如 5 秒）可能因网络波动误判正常请求</li>
        <li>时间太长（如 120 秒）又会拖慢系统响应</li>
        <li>30 秒是通用的默认值，但你可以根据实际场景调整</li>
      </ul>
    </li>
  </ul>

  <div className="api-tip">
    <p><strong>超时时间设置建议：</strong></p>
    <ul style={{ marginTop: '8px', paddingLeft: '20px' }}>
      <li>如果对方系统响应较慢（比如需要查询大量历史数据），可以适当调长超时时间（如 60 秒）</li>
      <li>如果是实时性要求高的请求（如支付、订单接口），建议调短超时时间（如 10-15 秒），避免用户等待过久</li>
    </ul>
  </div>
</div>
